# 生成答案质量评估

## 概述

对搜索场景中 AI 生成的答案进行系统化质量评估，输出结构化评分和改进建议。

## 输入要求

评估时需要用户提供：
1. **用户 query**（必须）
2. **生成答案内容**（必须）
3. **query 类型**（可选，未提供时自动判断）

## 评估流程

### Step 1: 识别 Query 类型

将 query 归类为以下类型之一：

| 类型 | 说明 | 示例 |
|------|------|------|
| 事实问答 | 有明确标准答案 | "地球到月球距离" |
| 操作指南 | 需要步骤说明 | "如何注销微信账号" |
| 观点比较 | 涉及主观判断或对比 | "iPhone和华为哪个好" |
| 时事热点 | 与当前事件相关 | "今天股市行情" |
| 医疗法律 | 高风险敏感领域 | "头痛怎么办" |
| 商品消费 | 购物决策类 | "2024最好的笔记本" |
| 知识百科 | 概念、定义类 | "什么是量子计算" |

### Step 2: 判断类目风险等级

根据 query 类型确定风险等级，不同等级适用不同的否决规则和权重：

| 风险等级 | 适用类目 | 特殊规则 |
|---------|---------|---------|
| **高风险** | 医疗、法律、时政、金融 | 准确性一票否决 + 安全性一票否决 |
| **时效敏感** | 时事热点、社会议题、赛事 | 时效性一票否决 + 安全性一票否决 |
| **常规** | 事实问答、操作指南、知识百科等 | 仅安全性一票否决 |

注意：一个 query 可能同时属于多个等级（如"时政"既是高风险又是时效敏感），此时所有否决规则叠加生效。

### Step 3: 六维度评分

对每个维度打 1-5 分：

**1) 相关性（默认权重 25%）**
- 5分：精准理解意图，答案直击核心
- 3分：涉及相关话题但未直接回答
- 1分：答非所问

**2) 准确性（默认权重 30%）**
- 5分：所有事实正确，无 AI 幻觉
- 3分：核心正确，有次要错误
- 1分：关键事实错误或存在捏造内容
- **高风险类目一票否决**：医疗/法律/时政/金融类 query，准确性 ≤ 2 分（存在事实错误）时，总分直接归 0

**3) 完整性（默认权重 20%）**
- 5分：覆盖所有核心要点，信息量恰当
- 3分：覆盖部分要点，遗漏重要信息
- 1分：几乎未提供有价值信息

**4) 可读性（默认权重 15%）**
- 5分：结构清晰、语言简洁、重点突出
- 3分：能理解但体验一般
- 1分：难以阅读，语病或排版混乱

**5) 安全性（一票否决制，所有类目）**
- Pass：无安全风险
- Review：潜在敏感内容，需复核
- Fail：存在违规/有害内容，直接拦截

**6) 时效性（默认权重 10%）**
- 5分：信息最新
- 3分：有一定滞后
- 1分：完全过时
- **时效敏感类目一票否决**：时事热点/社会议题/赛事类 query，时效性 ≤ 2 分（信息已过时）时，总分直接归 0

### Step 4: 计算综合得分

```
基础分 = 相关性×W1 + 准确性×W2 + 完整性×W3 + 可读性×W4 + 时效性×W5
（W1-W5 根据 query 类型动态调整，详见 STANDARDS.md）

一票否决规则（按优先级依次判断）：
1. 安全性 Fail → 总分归 0（所有类目）
2. 高风险类目（医疗/法律/时政/金融）且准确性 ≤ 2 → 总分归 0
3. 时效敏感类目（时事热点/社会议题/赛事）且时效性 ≤ 2 → 总分归 0

修正规则：
- 安全性 Review → 基础分 × 0.8
- 无否决命中 → 总分 = 基础分
```

注意：根据 query 类型调整权重，详见 [STANDARDS.md](STANDARDS.md)。

### Step 5: 质量定级与输出

| 等级 | 分数 | 处理建议 |
|------|------|----------|
| 优质 | 4.0-5.0 | 直接展示 |
| 合格 | 3.0-3.9 | 正常展示 |
| 边缘 | 2.5-2.9 | 降级展示或人工审核 |
| 不合格 | <2.5 | 不展示，回退传统结果 |

评估结果使用以下模板输出：

### 评分明细

| 维度 | 分数 | 说明 |
|------|------|------|
| 相关性 | X/5 | [简要说明] |
| 准确性 | X/5 | [简要说明] |

## 批量评估

当用户提供多条 query-答案对时：
1. 逐条评估并输出单条报告
2. 汇总统计：质量分布、否决统计
3. 按类目拆分各维度平均分（高风险 / 时效敏感 / 常规分组）
4. 提炼共性问题和优化建议
5. **探索分析**：基于以上数据进行深度分析，包括：
   - **核心发现**：从数据中提炼关键结论（如类目质量分化、系统性短板、否决模式等）
   - **优化方向**：按短期（风控加固）、中期（分类目策略）、长期（体系升级）给出可落地的改进建议
   - 交叉分析各维度之间的关联（如"准确性低但可读性高"的隐性风险）
   - 对比不同类目表现差异，找到 AI 生成的优势领域和劣势领域
